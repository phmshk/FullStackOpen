```mermaid
sequenceDiagram
  participant browser
  participant server

    browser->>server: POST  https://studies.cs.helsinki.fi/exampleapp/new_note
    activate server
    server-->>browser: HTTP status code 302
    deactivate server

    Note right of browser: POST request to the server address new_note. The server responds with HTTP status code 302.

    browser->>server: GET https://studies.cs.helsinki.fi/exampleapp/notes
    activate server
    server-->>browser: HTML document
    deactivate server

    Note right of browser: GET request to the address defined in the header's Location. Reload of page.

    browser->>server: GET https://studies.cs.helsinki.fi/exampleapp/main.css
    activate server
    server-->>browser: CSS file
    deactivate server

    Note right of browser: Reload causes three more GET requests. One for CSS file.

    browser->>server: GET https://studies.cs.helsinki.fi/exampleapp/main.js
    activate server
    server-->>browser: JS file
    deactivate server

    Note right of browser: One for JS file. The browser starts executing the JavaScript code that fetches the JSON from the server.

    browser->>server: GET https://studies.cs.helsinki.fi/exampleapp/data.json
    activate server
    server-->>browser: [*JSON of current objects containing messages from the page in form of { content: "hello world", date: "2025-05-20T11:22:17.590Z" }*]
    deactivate server

    Note right of browser: The browser executes the callback function that renders the notes.
```
